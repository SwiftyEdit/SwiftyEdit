# Das Templatesystem
Als Templatesystem kommt smarty zum Einsatz.
Wer ein eigenes Template erstellen möchte, sollte sich also zunächst mit der smarty Dokumentation
vertraut machen.

## Gut zu wissen

* Alle Themes befinden sich im Verzeichnis `/public/assets/themes/`.
* Die Themes `administration` und `default` werden bei jedem Update überschrieben.
* Themes können den Funktionsumfang von SwiftyEdit erweitern

## Verzeichnisstruktur

* themes/theme-name/
    * dist/
        * css/
        * js/
    * docs/
    * php/
    * templates/
    * templates-mail/
    * info.xml

### Zusätzliche Daten schreiben (PHP)

#### themes/theme-name/php/page-values.php

Über diese Datei wird der Seiten-Editor um einen **"Theme"-Tab** erweitert. 
Dieser erscheint, sobald eine Seite mit dem Theme `theme-name`gespeichert wurde.
Die Daten werden in der Spalte `page_template_values` als JSON gespeichert.
Ausgelesen werden die Daten in der Variable `$get_page['page_template_values']`. 

Über den Variablennamen `theme_values['..']` werden Daten übergeben und automatisch
in `page_template_values` gespeichert.

```PHP
<?php
// themes/default/php/page-values.php
global $get_page;
echo '<h5>Theme Values <small>(Theme: theme-name)</small></h5>';
$page_template_data = json_decode($get_page['page_template_values'],true);
$page_template_teaser = html_entity_decode($page_template_data['teaser_text'], ENT_QUOTES | ENT_XML1, 'UTF-8');
$input_wysiwyg_page_teaser = [
    "input_name" => "theme_values[teaser_text]",
    "input_value" => $page_template_teaser,
    "label" => 'Teaser Text',
    "type" => "textarea",
    "mode" => "wysiwyg"
];
echo se_print_form_input($input_wysiwyg_page_teaser);
```

#### themes/theme-name/php/category-values.php

Analog zur Seiten-Erweiterung wird hier der **Kategorie-Editor** um einen
"Theme"-Tab erweitert.
Die Daten werden in der Spalte `cat_template_values` als JSON gespeichert.
Ausgelesen werden die Daten in der Variable `$get_category['cat_template_values']`.

Über den Variablennamen `theme_values['..']` werden Daten übergeben und automatisch
in `cat_template_values` gespeichert.

```PHP
<?php
// themes/theme-name/php/category-values.php
global $get_category;
$category_template_data = json_decode($get_category['cat_template_values'],true);
echo '<h5>Theme Values <small>(Theme: theme-name)</small></h5>';
$category_teaser = html_entity_decode($category_template_data['teaser_text'], ENT_QUOTES | ENT_XML1, 'UTF-8');
$input_wysiwyg_page_teaser = [
    "input_name" => "theme_values[teaser_text]",
    "input_value" => $category_teaser,
    "label" => 'Teaser Text',
    "type" => "textarea",
    "mode" => "wysiwyg"
];
echo se_print_form_input($input_wysiwyg_page_teaser);
```

## Besonderheiten
Ein Template ist nicht nur für das Aussehen Deiner Website verantwortlich,
sondern kann auch die Optik bzw. die Funktionen des Editors tinyMCE steuern.
Damit kannst Du erreichen, dass sich die WYSIWYG Ansicht so nah als möglich am Original-Layout
(also dem Frontend) orientiert.

__Zwei Dateien sind dafür maßgeblich verantwortlich:__

* themes/deinTemplate/dist/css/editor.css
* themes/deinTemplate/dist/tinyMCE_config.js

Die CSS Datei ist für das Aussehen der Inhalte im Editor verantwortlich.
Dies ist der einfachste Weg, den Editor an Dein Layout anzupassen.
Die JavaScript Datei ist für die Konfiguration zuständig.
Weitere Informationen findest Du in der tinyMCE Dokumentation.
